🎯 Hoje vamos além

Até agora trabalhamos em uma única linha do tempo. Hoje você aprende a criar ramificações paralelas — essencial para o trabalho em equipe.

O Problema

Todos no main = caos

Imagine o projeto ODS com 5 pessoas trabalhando diretamente no main:

git log — main sem branches
a1b2c3 Pedro: muda o CSS do header
e4f5g6 Ana: adiciona mapa (incompleto)
7h8i9j Pedro: reverte o CSS (quebrou)
k1l2m3 Ana: mapa quebrado, revert
n4o5p6 Carlos: commit por cima do revert
  • Funcionalidades incompletas entram no código principal e quebram o projeto
  • Impossível trabalhar em paralelo sem pisar no código do colega
  • O histórico vira um labirinto impossível de entender
  • Um erro de uma pessoa para o trabalho de todos
💡 A solução

Cada pessoa (ou funcionalidade) tem a sua própria branch. O main só recebe código pronto e revisado.

Conceito

Branch = linha do tempo alternativa

Uma branch é um ponteiro para um commit específico. Criar uma branch não copia código — é apenas um novo caminho a partir de um ponto.

●──●──●──────────────────────────● ← main (HEAD)
│ (branch criada aqui)
└──●──●──● ← feature/mapa-ods
  • A branch main (ou master) é a linha principal e estável do projeto
  • Você cria branches para funcionalidades, correções ou experimentos
  • Cada branch evolui independentemente até ser integrada de volta
🌿 Por que Git é tão rápido?

Branches no Git são apenas um arquivo de 41 bytes (o hash do commit). Criar uma branch é instantâneo, diferente de outros sistemas que copiavam arquivos.

Comandos

git branch — Criar e gerenciar

bash — git branch
$git branch
* main
  feature/mapa-ods
  fix/corrige-login
# Asterisco (*) indica a branch atual
$git branch feature/nova-funcionalidade
# Cria a branch (mas NÃO muda para ela)
$git branch -d feature/nova-funcionalidade
Deleted branch feature/nova-funcionalidade (was a1b2c3d).
# -d deleta branch já mesclada · -D força deleção
$git branch -a
# Lista todas, incluindo branches remotas
⚠️ -d vs -D

Use -d (minúsculo) para deletar branches já mescladas — o Git avisa se você tentar apagar algo não integrado. Use -D somente se tiver certeza de que não precisa mais dos commits.

Navegação

git switch — Mudar de branch

Comando moderno (Git 2.23+):

git switch
$git switch feature/mapa-ods
Switched to branch 'feature/mapa-ods'
$git switch -c feature/nova
Switched to a new branch 'feature/nova'
# -c = criar E mudar em um só comando
$git switch main
Switched to branch 'main'
$git switch -
# Volta para a branch anterior (atalho útil!)

Comando legado (ainda funciona):

git checkout (legado)
$git checkout feature/mapa-ods
$git checkout -b feature/nova
# Equivalente a switch -c
💡 Use git switch

git checkout é legado e faz coisas demais (muda branches E restaura arquivos). Prefira git switch para branches e git restore para arquivos.

Feature Branch Flow

O fluxo completo de uma feature

01
git switch -c feature/mapa-ods
Cria e já muda para a nova branch a partir do main atual.
02
git add . && git commit -m "feat: adiciona mapa base"
Trabalha e cria commits semânticos normalmente na sua branch.
03
git switch main
Volta para main quando a funcionalidade estiver pronta e testada.
04
git merge feature/mapa-ods
Incorpora os commits da feature branch no main.
05
git branch -d feature/mapa-ods
Limpa a branch após o merge (opcional, mas recomendado).
🔄 Na prática com GitHub

No projeto ODS, o merge não vai acontecer localmente assim — você vai abrir um Pull Request para que um colega revise o código. Veremos isso na Aula 3.

Integrando Branches

git merge — Fast-forward

Acontece quando main não teve nenhum commit novo desde que a branch foi criada. O Git simplesmente "avança" o ponteiro.

Antes:
●──● (main)
└──●──● (feature)

Depois do merge:
●──●──●──● (main + feature, sem merge commit)
fast-forward merge
$git switch main
Switched to branch 'main'
$git merge feature/mapa-ods
Updating a1b2c3d..e4f5g6h
Fast-forward
 mapa.html | 42 +++++++++++++
✅ Mais limpo

O fast-forward gera um histórico linear, sem commits de merge extras. Ideal para features pequenas e isoladas.

Integrando Branches

git merge — Merge commit

Acontece quando main também teve commits enquanto a feature estava sendo desenvolvida. O Git cria um commit de merge para unir as duas histórias.

Situação:
●──●──● (main com novos commits)
└──●──● (feature)

Após merge:
●──●──●────● (merge commit)
└──●──●──/
merge commit
$git merge feature/mapa-ods
Merge made by the 'ort' strategy.
 mapa.html | 42 ++++++++++
$git log --oneline
c3f9a1b Merge branch 'feature/mapa-ods'
8d2e104 fix: corrige api de metas
5a7b3c9 feat: mapa interativo
💡 Conflito possível

Se as duas branches alteraram o mesmo trecho de um arquivo, o Git vai pausar e pedir que você resolva o conflito manualmente.

Resolvendo Conflitos

Conflitos: o Git pedindo ajuda

Um conflito acontece quando duas branches alteraram as mesmas linhas de um arquivo. O Git não sabe qual versão é a "certa" — você decide.

O Git marca o arquivo com delimitadores de conflito:

<!-- index.html -->

<<<<<<< HEAD
  <h1>Página Inicial do Projeto ODS</h1>
=======
  <h1>Bem-vindo ao Projeto de Sustentabilidade</h1>
>>>>>>> feature/titulo-pagina
❌ <<<<<<< HEAD

Código da branch atual (main)

── =======

Separador entre as versões conflitantes

✎ >>>>>>>

Código da branch que está sendo mesclada

Passo a Passo

Como resolver um conflito

01
git status
Identifica quais arquivos estão em conflito — marcados como "both modified".
02
Abra o arquivo no VS Code
O editor destaca visualmente os conflitos com botões "Accept Current", "Accept Incoming" ou "Accept Both".
03
Edite manualmente
Decida qual código manter. Remova os marcadores (<<<, ===, >>>) e deixe apenas o código final correto.
04
git add index.html
Marca o conflito como resolvido adicionando o arquivo à Staging Area.
05
git commit -m "fix: resolve conflito no título"
Finaliza o merge com um commit descritivo.
🛡️ Dica profissional

Faça git pull ou git merge main na sua feature branch com frequência para pegar as atualizações do main cedo, enquanto os conflitos são pequenos e fáceis de resolver.

O comando mais poderoso para entender o histórico do projeto:

|/
*   8d2e104 Merge branch 'feature/auth'
|\

🔧 Alias útil

Configure um atalho: git config --global alias.graph "log --oneline --graph --all --decorate" — depois é só usar git graph.
